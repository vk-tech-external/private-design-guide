|===
|Термин
|Определение

|Балансировщик нагрузки (Loadbalancer)
|Балансировщики нагрузки позволяют распределять поток запросов (или сетевой трафик) между несколькими устройствами (например, серверами) с целью:

* Горизонтального масштабирования.
* Обеспечения отказоустойчивости (резервирования).
* Оптимизации использования ресурсов (за счёт динамического добавления или удаления устройств в зависимости от нагрузки)

|Вертикальное масштабирование
|Увеличение производительности системы за счёт наращивания мощности отдельных её компонентов

|Вес инстанса при балансировке
|Доля запросов (сетевого трафика), которые будут передаваться на устройство. Чем больше вес устройства, тем больше запросов (трафика) будет распределяться балансировщиков на это устройство

|Виртуальная сеть (Virtual network)
|Сеть, созданная при помощи виртуализации (виртуального маршрутизатора). Работает аналогично традиционной сети. Обеспечивает связь между виртуальными машинами (ВМ) и другими устройствами

|Виртуальный маршрутизатор
|Программная реализация сетевого маршрутизатора, которая работает аналогично традиционному маршрутизатору, но запускается при помощи средств виртуализации

|Виртуальный порт (Virtual port)
|Виртуальная точка подключения виртуального сетевого интерфейса к виртуальной сети (подсети). Работает аналогично традиционному порту

|Вычислительный узел (Compute node), Гипервизор (Hypervisor)
|Тип узла, предоставляющий вычислительные ресурсы (vCPU) и оперативную память (RAM) для ВМ

<!--- //Каждый гипервизор имеет установленную на жёсткие диски ОС, настроенный гипервизор KVM, настроенные модули поддержки SDS Ceph и (или) СХД, и агенты для работы с сетевыми настройками -->

|Горизонтальное масштабирование
|Увеличение производительности за счёт разделения задач (и параллельного их выполнения) на несколько серверов

|Группа безопасности (Security Group)
|Наборы настраиваемых разрешающих правил, которые регулируют права сетевого доступа (по определенному протоколу, через конкретный порт, для определенных IP-адресов или групп безопасности)

|Группа узлов (Node group)
|Множество узлов с общим типом ВМ (flavor)

|Деплой-нода, узел развёртывания (deploy node)
|Вспомогательный узел для развёртывания и конфигурирования {var(sys2)}. Обычно включает в себя репозиторий, в котором хранятся все артефакты, необходимые для процессов установки и обновления {var(sys2)}

|Диск (Volume)
|Сетевое блочное устройство, которое обеспечивает хранилище данных для ВМ. Все диски в {var(sys6)} являются сетевыми и защищены репликацией данных, обеспечивающей надежность хранения и отказоустойчивость

|Журнал серийной консоли
|Способ получить удалённый доступ к серверу вне зависимости от состояния сети или операционной системы. Доступ осуществляется через систему удалённого доступа VNC (Virtual Network Computing)

|Зона доступности (Availability Zone)
|Логическое деление и объединение гипервизоров для физической их изоляции и обеспечения отказоустойчивости

|Инстанс (Instance), ВМ
|Виртуальная машина, которая запускается и работает в облаке

|Кластер Kubernetes
|Кластер Kubernetes состоит из набора машин, так называемых узлов (nodes), которые запускают контейнеризированные приложения. Кластер должен иметь как минимум один рабочий узел

|Ключевая пара (Key Pair)
|Взаимосвязанная пара, состоящая из открытого и закрытого асимметричных ключей.

Открытый ключ используется для шифрования данных, может быть широко распространён.

Закрытый ключ используется для расшифровки данных, хранится только у владельца ключевой пары

|Метод балансировки
|Метод распределения заданий между несколькими сетевыми устройствами (например, серверами) с целью оптимизации использования ресурсов

|Образ, образ диска (Image)
|Файл, несущий в себе полную копию содержимого диска: структуру файловой системы и данные, находящиеся на диске. Образы используются для создания не пустых дисков, в том числе загрузочных.

Обычно образ, используемый в {var(sys3)} при создании загрузочного диска ВМ, содержит установленную операционную систему, а иногда и дополнительное ПО. Имя образа, чаще всего, отражает тип и версию операционной системы и другие особенности содержания образа

|Под (Pod)
|Группа из одного или нескольких контейнеров приложений, включающая общие используемые хранилища (тома), IP-адрес и информацию по их запуску

|Правило группы безопасности
|Набор параметров, определяющих условия прохождения трафика

|Резервная копия (Backup)
|Копия (ВМ или диска) в том состоянии, когда была создана точка восстановления

|Снимок диска, снэпшот (snapshot)
|Состояние файловой системы на определенный момент времени.

Создание снимка диска останавливает запись гостевой ОС на короткое время, сохраняет состояние файлов системы и все последующие изменения записывает в отдельный файл

|Статическая маршрутизация
|Способ направления трафика из подсети на заданные диапазоны IP-адресов через ВМ, которые указаны в качестве промежуточных узлов

|Теги инстанса (Tag)
|Произвольные сочетания символов, позволяющие индексировать инстансы или фильтровать их

|Тип ВМ (Flavor)
|Конфигурация инстанса, определяющая характеристики его ресурсов. Тип ВМ определяет количество и тип CPU, объём RAM и некоторые другие параметры ВМ

|Топология сети
|Графическое представление расположения и соединения сетевых устройств:

* Сетей.
* Подсетей.
* Маршрутизаторов.
* ВМ

|Узел (Node)
|Отдельная физическая или виртуальная машина, на которой развёрнуты и выполняются компоненты {var(sys2)}

|Управляющий узел (Controller node)
|Тип узла, обеспечивающий управление {var(sys5)} и автоматизацию множества процессов её работы, но не предоставляющий ресурсы для конечных пользователей. Управляющие узлы предоставляют API и другие интерфейсы для работы с {var(sys5)}, обеспечивают работу внутренних сервисов {var(sys2)} и планировщиков, а также некоторые инструменты для администрирования {var(sys2)}

|Файловое хранилище (File storage)
|Виртуальная файловая система, которую можно подключать к ВМ

|ЦОД
|Центр обработки данных — специализированное здание или помещение для размещения серверного и сетевого оборудования, оснащённое всей необходимой инфраструктурой, например, электропитание, охлаждение, сети связи и прочее.

|CEPH
|Программная объектная отказоустойчивая сеть хранения данных

|Helm
|Менеджер пакетов для Kubernetes, который используется для быстрой установки сложных приложений, например, CRM, e-commerce, БД.

|JSON
|Текстовый формат обмена данными, основанный на JavaScript

|Kubectl
|Инструмент командной строки для управления кластерами Kubernetes. Kubectl используется для развёртывания приложений, проверки ресурсов кластера, а также просмотра логов

|Kubernetes Dashboard
|Универсальный веб-интерфейс для кластеров Kubernetes. Он позволяет пользователям управлять приложениями, работающими в кластере, и устранять их неполадки, а также управлять самим кластером

|Master-узел
|Главный узел, который отвечает за управление кластером Kubernetes

|Root-диск, загрузочный диск
|Основной диск, с которого запускается операционная система при старте ВМ

|YAML
|Язык сериализации данных, обычно используемый в файлах конфигурации, таких как Kubernetes, Docker, Ansible и других
